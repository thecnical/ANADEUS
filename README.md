## рҹ§  What is ANADEUS?

**ANADEUS** is not just another scanner вҖ” it's an **AI cybersecurity brain** that thinks, adapts, and executes like a senior penetration tester.

It orchestrates **5 autonomous agents** across a complete attack pipeline, powered by LLM-driven decision-making. From subdomain discovery to a polished vulnerability report with PoC evidence вҖ” one command does it all.

> **Built for:** Security researchers В· Penetration testers В· Bug bounty hunters В· Red teams

## вңЁ Features

### рҹӨ– Multi-Agent Autonomous Pipeline

| Agent | What It Does |
|---|---|
| **рҹ”Қ Recon Agent** | Subdomain enumeration, alive host detection, technology fingerprinting |
| **рҹ“Ў Scanner Agent** | Port/service scanning, directory fuzzing, endpoint discovery, web server analysis |
| **рҹ§Ә Analysis Agent** | AI + heuristic vulnerability candidate identification and prioritization |
| **рҹ’Ҙ Exploit Agent** | Safe validation of SQLi, XSS, IDOR, CSRF, Auth Bypass вҖ” zero destructive payloads |
| **рҹ“„ Report Agent** | Impact assessment, confidence scoring, PoC markdown, final report generation |

### рҹ§  AI Decision Engine

- **Dual-provider architecture** вҖ” Bytez (primary) + OpenRouter (fallback)
- Intelligent phase selection based on context quality and completed work
- Automatic confidence calibration and scoring
- Context-aware prompt generation for each phase

### рҹӣ пёҸ 14+ Integrated Security Tools

```
Subfinder  В·  Amass  В·  Assetfinder  В·  httpx  В·  WhatWeb
Nmap  В·  ffuf  В·  dirsearch  В·  Feroxbuster  В·  Nikto
WebProbe  В·  SocketProbe  В·  RouteProbe  В·  and more...
```

> Every tool has a **smart fallback chain** вҖ” if `nmap` fails, `socketprobe` takes over instantly. No phase ever silently dies.

### рҹ”җ Production-Grade Reliability

- **Strict tool option sanitization** вҖ” per-tool schemas prevent option leakage across fallback chains
- **Deterministic state management** вҖ” atomic writes, hard resets, single source of truth
- **Early CLI injection prevention** вҖ” malicious targets rejected before anything executes
- **Degraded scan recovery** вҖ” empty results trigger automatic retry with alternate tools

## рҹҡҖ Quick Start

### Prerequisites

| Requirement | Version | Why |
|---|---|---|
| **Node.js** | вүҘ 20.x | Core runtime |
| **Python** | вүҘ 3.9 | Tool runner |
| **Kali Linux** | 2024.x+ | Pre-installed security tools |

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/thecnical/ANADEUS.git
cd ANADEUS

# 2. Install Node.js dependencies (CRITICAL: Required for CLI)
npm install

# 3. Install Python dependencies (with Kali Linux fix)
pip install -r requirements.txt --break-system-packages

# 4. Configure API keys
cp .env.example .env
# вҶ’ Edit .env with your keys (see API Configuration below)

# 5. Verify installation
npm run banner
```

> **Note:** Nmap, Subfinder, ffuf, Nikto, etc. come pre-installed on Kali Linux. On other distros, install them via your package manager.

## рҹ”‘ API Configuration

ANADEUS uses a **dual-provider AI architecture** with automatic fallback.

### Create Your `.env` File

```bash
cp .env.example .env
```

```env
# в”Җв”Җ PRIMARY: Bytez в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

BYTEZ_MODEL=your_preferred_model

# в”Җв”Җ FALLBACK: OpenRouter в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

OPENROUTER_MODEL=openai/gpt-4o-mini
```

### Where to Get Keys

| Provider | Link | Notes |
|---|---|---|
| **Bytez** | [bytez.com](https://bytez.com) | Primary вҖ” fast, dedicated infrastructure |
| **OpenRouter** | [openrouter.ai](https://openrouter.ai) | Fallback вҖ” 200+ models, pay-per-token |

> рҹ’Ў You only need **one** provider to get started. Both configured? Bytez runs first, OpenRouter kicks in automatically on failure.

ANADEUS auto-loads `.env` at startup вҖ” no extra setup needed.

## рҹ’» Usage & Commands

### рҹӨ– Full Auto Mode *(Recommended)*

Let the AI brain run the entire pipeline:

```bash
# Deep scan вҖ” full pipeline, thorough analysis
node src/index.js auto --target example.com --deep

# Light scan вҖ” faster, reduced scope
node src/index.js auto --target example.com --mode light
```

### рҹ’¬ Interactive Console

If you want the terminal to stay open so you can type multiple commands, start the interactive chat mode:

```bash
node src/index.js
# or
node src/index.js chat
```

Then you can type commands directly into the ANADEUS prompt:
```text
> @recon example.com
> @scan example.com
> /show recon
```

### рҹҺҜ Single-Shot Commands (Exits when done)

Run specific phases manually from your normal terminal:

```bash
@recon example.com          # Subdomain enum + alive detection + tech fingerprinting
@scan example.com           # Port scan + directory fuzzing + web server analysis
@analysis example.com       # AI vulnerability identification + prioritization
@exploit example.com        # Safe PoC validation (SQLi, XSS, IDOR, CSRF, Auth)
@report example.com         # Impact assessment + final markdown report
```

Via CLI:

```bash
node src/index.js chat --message "@recon example.com"
node src/index.js chat --message "@scan example.com"
node src/index.js chat --message "@analysis example.com"
node src/index.js chat --message "@exploit example.com"
node src/index.js chat --message "@report example.com"
```

### рҹ•№пёҸ Agent Mode

```bash
node src/index.js agent recon example.com
node src/index.js agent scan example.com
node src/index.js agent recon example.com --json     # Structured JSON output
```
